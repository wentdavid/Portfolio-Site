Angular Movie library

This project is an Angular-based client-side application for the movie API. It provides users with the ability to browse movies, get information about their favorite movies, and manage their profiles. Feel free to test it here.

Purpose & Context

The purpose of the myFlix-Angular app was to create a full-stack project that showcases expertise in Angular, Node.js, Express, and TypeScript, while offering a user-friendly platform for movie enthusiasts to explore and manage their favorite films. Another goal was to reuse the Movie API that was previously created and integrate it into different frameworks.

Objective

The primary objective of the project was to build a complete, responsive, and user-friendly application with Angular that allows users to explore and manage their favorite movies while demonstrating my skills in full-stack development.

Approach

  1. Project Dependencies: The project utilized Node.js, Angular CLI, Angular Material, Angular Flex Layout, and TypeScript for development.
  2. Code Organization:The code was organized into modules and components, providing a clear structure and separation of concerns.
  3. Server-Side Management: A RESTful API that was previously developed using Node.js and Express to handle server-side tasks and interact with the movie database.
  4. Client-Side Management: Angular was used to build a responsive and user-friendly single-page application, with components like user registration, login, movie cards, and profile management.

Challenges

The primary challenge in building the myFlix-Angular-client was mastering the Angular framework, integrating it with the server-side, and creating a seamless user experience. By dedicating time to learning Angular concepts and best practices, I was able to overcome these challenges and create a highly functional application.

Duration

The development process took less time than initially anticipated due to the understanding already gained from the previous React Movie app. Although the learning curve associated with Angular and its integration with the back-end is higher, the structured approach offered by Angular was appreciated. The time invested allowed for a solid understanding of the framework and its various features, resulting in a polished final product.

Project Highlights

The following API endpoints are used by this service:

  • Modular code organization, separating concerns into components and modules for better maintainability and scalability.
  • Responsive design using Angular Material and Angular Flex Layout, ensuring a seamless user experience across devices.
  • Robust API integration, utilizing endpoints for user registration, login, movie details retrieval, and profile management.
  • Implementation of user authentication and authorization with JWT authentication for secure user access.

Conclusion

The myFlix-Angular project showcases the potential of Angular as a powerful client-side framework for building responsive and user-friendly web applications. The final product demonstrates my expertise in full-stack development, particularly with Angular, Node.js, and Express. The successful completion of this project not only serves as an addition to my professional portfolio but also as a testament to the importance of dedication and continuous learning in the field of web development.

Future Improvements

There are several potential improvements that can be made to the myFlix-Angular app in the future to enhance its functionality and user experience:

  1. Advanced Filtering and Sorting: Implement more advanced filtering and sorting options for users to easily find movies based on specific criteria such as release year, ratings, or language.
  2. Social Features:Integrate social features like user reviews, comments, or recommendations, allowing users to share their opinions and favorite movies with others.
  3. Mobile App: Develop a native mobile app for Android and iOS platforms, providing an even more convenient and accessible experience for users on the go.
  4. Performance Optimization: Continuously monitor the app's performance and optimize it for faster load times and smoother user experience, particularly on slower connections or devices.
  5. Internationalization: Add support for multiple languages, making the app accessible to a broader audience and promoting cultural diversity.
  6. Personalized Content: Implement personalized content recommendations based on user preferences and browsing history, providing a more tailored experience for each user.
  7. Enhanced Security: Continuously review and update security measures to protect user data and privacy, ensuring a safe and secure platform for movie enthusiasts.

By implementing these future improvements, the myFlix-Angular app can continue to evolve and offer an even better experience for its users while showcasing the latest advancements in full-stack web development.
